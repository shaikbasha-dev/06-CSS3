# Security Policy

## Supported Versions

The latest version of this repository is actively maintained and updated.

| Version | Supported |
|----------|-----------|
| Latest   | ✅ Yes |
| Older Versions | ❌ No |

---

# Reporting a Security Vulnerability

If you discover a security vulnerability related to this repository, please report it responsibly.

Instead of creating a public issue, please contact the repository maintainer through GitHub.

When reporting a vulnerability, include:

- A clear description of the issue.
- Steps to reproduce the problem.
- The affected files or examples.
- Any suggested mitigation or solution.

---

# Response Process

Every reported vulnerability will be:

- Reviewed promptly.
- Investigated carefully.
- Addressed when applicable.
- Resolved in future repository updates.

Valid reports will be acknowledged after verification.

---

# Responsible Disclosure

Please do not publicly disclose potential security issues until they have been reviewed.

Responsible disclosure helps protect repository users and allows sufficient time to evaluate and resolve reported concerns.

---

# Scope

This security policy applies to:

- HTML examples
- CSS examples
- Documentation
- Repository configuration
- Supporting educational resources

---

# Thank You

Thank you for helping keep this repository secure and reliable for learners, educators, and developers.
